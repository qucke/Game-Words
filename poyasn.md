# Архитектура проекта
![](Game_words.png)
## Класс `Cell` (Клетка)

**Суть:**  
Элемент игрового поля

### Свойства

- `letter: char` — символ, хранящийся в ячейке.

### Методы

- `isEmpty()` — проверяет, пуста ли клетка.
- `getLetter()` / `setLetter()` — управление содержимым клетки.

---

## Класс `Player` (Игрок)

**Суть:**  
Хранилище данных об игроке и его текущем прогрессе

### Свойства

- `name: string` — имя участника.
- `score: int` — накопленные баллы.
- `consecutiveSkips: int` — количество пропущенных ходов подряд.

### Методы

- `addScore(int)` — обновление счета игрока.
- `skipTurn()` / `resetSkips()` — управление счетчиком пропусков.

---

## Класс `Dictionary` (Словарь)

**Суть:**  
Обеспечение правил игры через проверку слов.

### Свойства

- `validWords` — множество всех доступных слов из файла.
- `usedWords` — множество уже сыгранных слов (для исключения повторов).

### Методы

- `isValidWord(string)` — проверка наличия слова в словаре.
- `isWordUsed(string)` — проверка, использовалось ли слово ранее.

---

## Класс `Board` (Игровое поле)

**Суть:**  
Управление логикой сетки `5x5` и алгоритмом поиска слов.

### Свойства

- `grid: vector<vector<Cell>>` — двумерный массив клеток.

### Методы

- `initCenterWord(string)` — установка стартового слова в центр.
- `placeLetter(int, int, char)` — размещение буквы на поле.
- `canFormWord(string, ...)` — алгоритм DFS (поиск в глубину) для проверки возможности составить слово змейкой.
- `display()` — отрисовка состояния поля в консоли.

---

## Класс `Game` (Контроллер)

**Суть:**  
Центральный узел, управляющий игровым циклом.

### Свойства

- Объекты `Board`, `Dictionary` и массив объектов `Player`.
- `currentPlayerIndex` — текущий ход игрока.

### Методы

- `play()` — главный цикл игры, в котором происходит ввод данных, проверка условий победы и смена игроков.
- `clearInputBuffer()` — служебный метод для корректной работы с потоком `cin`.

---

# Взаимодействие компонентов

## Композиция

Класс `Game` содержит в себе объекты `Board` и `Dictionary`, что означает, что они неразрывно связаны с жизненным циклом игры.

## Управление

`Game` получает данные от `Player` (ввод пользователя), запрашивает у `Board` проверку возможности хода, а у `Dictionary` — лексическую корректность слова.

## Алгоритм

Поиск слов на поле осуществляется через метод `dfs` внутри класса `Board`, что позволяет реализовать правила игры, где буквы должны быть смежными.
